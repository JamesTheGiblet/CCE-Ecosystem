CCE Node

Your Personal Trading Engine. Sovereign. Local. Deterministic.

CCE Node is the real engine ‚Äî a dedicated hardware device that runs the Cascade Compounding Engine locally, privately, and autonomously.

Node is the execution layer of the ecosystem.

Hardware Specifications

  • Model: Raspberry Pi 5 (8GB RAM)
  • Storage: 128GB NVMe SSD (Kingston)
  • Cooling: Official Pi 5 Active Cooler
  • Power: 27W USB-C (8-12W average draw)
  • OS: Debian 12 (Bookworm) 64-bit

Setup Process

1. Unbox and Connect

Connect Ethernet and power. Wait 60 seconds for boot.

2. Access Interface

Navigate to http://cce-node.local on your browser.

3. Configure Exchange

Enter your Binance API Key and Secret. Ensure "Spot & Margin Trading" is enabled, but "Withdrawals" are DISABLED.

4. Activate

Click "Start Engine". The Node will perform its first check within 6 hours.

Security

Your Binance API keys never leave Node.

  • Stored in encrypted SQLite database
  • Encrypted at rest using Fernet
  • Decrypted only in memory during API calls
  • No incoming connections from internet (local only)

Maintenance

Node is designed to run autonomously. You do not need to update software (it auto-updates) or restart the device.

Recommended: Log in quarterly to check logs and verify balance.

FAQ

Do I need to leave Node running 24/7?

Yes. Node checks market conditions every 6 hours. If it's offline, you might miss a transition.

What if my internet goes down?

Node will resume when connection is restored. Missed checks are logged.

Can I modify the strategy?

No. Node runs the CCE strategy only. It is a sovereign execution device for the CCE philosophy.

CCE Pocket

CCE in Your Pocket. Awareness, Clarity, Calm.

CCE Pocket is the mobile companion to Cloud and Node. It keeps you connected to your engine wherever you are.

Pocket is the presence layer of the ecosystem.

Features

  • Instant state notifications: Know when the engine transitions
  • Quick-view dashboard: Current state, position, performance
  • Market context: BTC price, Fear & Greed, trend at a glance
  • Peace of mind: Without opening Cloud or checking Node

How It Works

1. Connect

Pocket syncs with Cloud (automatic) or Node (via local API). Setup takes 30 seconds.

2. Receive Notifications

Get push alerts for state transitions, trades, and performance milestones. Frequency is low by design (1-5 per week).

3. Glance

Open the app to see the "Current State" card. No graphs, no complex charts. Just the numbers you need.

Notification Settings

Pocket lets you customise what you hear about:

  • State Changes: Default ON. Know when the regime changes.
  • Trades: Default ON. Know when execution happens.
  • Market Noise: Default OFF. You don't need hourly price alerts.

Privacy & Data

What Pocket Stores

  • Cloud account token (Cloud mode)
  • Node IP address (Node mode)
  • Notification preferences

What Pocket Does NOT Store

  • Binance API keys
  • Trading history
  • Personal financial information

FAQ

Can I use Pocket to control the engine?

No. Pocket is view-only. It does not let you start/stop the engine or place manual trades. Control happens in Cloud or Node.

Will Pocket drain my battery?

No. Pocket uses push notifications, not polling. Average battery impact is <1% per day.

What if I lose my phone?

Your engine keeps running. Pocket is just a view. It has no control over execution.
